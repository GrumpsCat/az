Describe Azure Identity, Access, and Security

Describe directory services in Azure

What directory services are available in Azure, and what roles do Microsoft Entra ID and Microsoft Entra Domain Services play?

  • Microsoft Entra ID is Azure’s cloud-based identity and access management service that manages users, groups, and applications, and supports authentication, authorization, and single sign-on.
  • Microsoft Entra Domain Services provides managed domain services such as domain join, group policy, and LDAP authentication without requiring customers to deploy and manage domain controllers.
  • Microsoft Entra ID is designed for modern cloud applications and identity federation, while Microsoft Entra Domain Services supports legacy applications that rely on traditional Active Directory features.
  • Both services help organizations centrally manage identities and control access to Azure resources and applications.

Learn more: Microsoft Learn – Directory services in Azure

Describe authentication methods

What authentication methods are available in Azure, including single sign-on (SSO), multifactor authentication (MFA), and passwordless authentication?

  • Authentication is the process of verifying the identity of a user, service, or device by requiring credentials to prove who they are.
  • Single sign-on (SSO) allows users to sign in once and access multiple applications and resources using a single identity.
  • Multifactor authentication (MFA) requires two or more forms of verification, significantly reducing the risk of credential-based attacks.
  • MFA factors include:
    • Something you know (password or PIN)
    • Something you have (phone or security key)
    • Something you are (fingerprint or facial scan)
  • Passwordless authentication removes passwords and uses stronger methods such as biometrics, trusted devices, or cryptographic keys.
  • Azure passwordless options include:
    • Windows Hello for Business
    • Microsoft Authenticator app
    • FIDO2 security keys
  • Microsoft Entra multifactor authentication is the service that enables and manages MFA methods in Azure.

Learn more: Microsoft Learn – Authentication methods in Azure

Describe external identities in Azure

What are external identities in Azure, and how are they used to grant access to users outside an organization?

  • An external identity is a user, device, or service that exists outside an organization’s Azure tenant.
  • Microsoft Entra External ID enables secure collaboration with external users such as partners, vendors, suppliers, and customers.
  • External users can bring their own identities, including corporate, government, or social identities (such as Google or Facebook).
  • The external user’s identity provider manages authentication, while access to applications and resources is controlled using Microsoft Entra ID or Azure AD B2C.
  • External identity capabilities include:
    • B2B collaboration – Allows guest users from other organizations to access enterprise applications and resources.
    • B2B direct connect – Establishes trust between Entra tenants for seamless collaboration, such as Teams shared channels.
    • Azure AD B2C – Manages identity and access for consumer-facing applications.
  • Guest access can be governed using access reviews, allowing organizations to periodically verify whether external users still require access.

Learn more: Microsoft Learn – External identities in Azure

Describe Microsoft Entra Conditional Access

What is Microsoft Entra Conditional Access, and how does it help enforce access policies based on identity signals?

  • Conditional Access is a policy-based tool in Microsoft Entra ID that controls access to resources based on identity-related signals.
  • Identity signals include:
    • Who the user is
    • Where the user is signing in from
    • What device is being used
    • Which application is being accessed
  • Conditional Access evaluates signals during sign-in and makes a decision to allow access, block access, or require multifactor authentication (MFA).
  • It enables adaptive security, such as requiring MFA only when a sign-in appears risky or comes from an unusual location.
  • Common Conditional Access use cases include:
    • Requiring MFA for administrators or remote users
    • Restricting access to approved client applications
    • Allowing access only from compliant or managed devices
    • Blocking access from untrusted or high-risk locations
  • Conditional Access balances security and productivity by enforcing protection only when needed.

Learn more: Microsoft Learn – Conditional Access
